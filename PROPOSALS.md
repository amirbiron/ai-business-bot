## 1. תמיכה בהודעות קוליות

> **הנמקה**: ישראלים שולחים הודעות קוליות בכמויות מטורפות. לקוח שנוהג ברכב או עסוק לא יקליד — הוא ילחץ על המיקרופון. כרגע הבוט מתעלם מהודעות קוליות לחלוטין, מה שיוצר חוויה שבורה.

**מה נדרש**:
- שימוש ב-OpenAI Whisper API לתמלול הודעה קולית לטקסט
- העברת הטקסט המתומלל לצינור RAG הקיים — ללא שינוי בליבה
- שליחת תשובה טקסטואלית רגילה (לא קולית)

**זרימה**:
```
לקוח: [שולח הודעה קולית: "היי, כמה עולה תספורת גברים?"]
בוט:   "שמעתי אתכם! 🎙️ תספורת גברים אצלנו עולה ..."
```

**מימוש טכני**:
- הוספת handler ל-`filters.VOICE` ב-`telegram_bot.py`
- הקובץ הקולי מורד מטלגרם (`.ogg`), נשלח ל-Whisper API, התמלול מועבר ל-`generate_answer`
- עלות Whisper: ~$0.006 לדקה — זניחה

**ערך ללקוח**: הבוט שלהם "מבין" גם כשהלקוחות מדברים, לא רק כשהם מקלידים. זה מרגיש כמו שירות אנושי אמיתי.

## 2. תמיכה בתמונות מלקוחות

> **הנמקה**: "אני רוצה תספורת כזו", "מה המוצר הזה?", "יש לי את הבעיה הזו בשיער" — לקוחות שולחים תמונות. בוט שמתעלם מתמונות מאבד הקשר חשוב.

**מה נדרש**:
- שימוש ב-GPT-4o (שכבר תומך בתמונות) לתיאור התמונה
- שילוב התיאור כחלק מהשאילתא ל-RAG
- אם אין טקסט נלווה, הבוט שואל "קיבלתי את התמונה! במה אוכל לעזור בקשר אליה?"

**תרחישים**:
| תרחיש | קלט | תגובת הבוט |
|--------|------|-------------|
| השראה | תמונה של תסרוקת | "תסרוקת יפה! אצלנו אפשר להשיג מראה דומה עם שירות בליאז'. רוצים לקבוע תור?" |
| מוצר | תמונה של מוצר | "זה נראה כמו שמפו של מותג X. אנחנו עובדים עם מוצרים דומים של..." |
| ללא הקשר | תמונה בלי טקסט | "קיבלתי את התמונה! איך אפשר לעזור?" |

**ערך ללקוח**: הבוט מרגיש חכם — בעל העסק יכול להתגאות ב"בוט שמבין תמונות".

## 3. מודעות לשעות פעילות וחגים

> **הנמקה**: לקוח שפונה בשבת בלילה צריך לקבל תגובה שונה מלקוח שפונה ביום שלישי בבוקר. כרגע הבוט מתנהג זהה בכל שעה.

**מה נדרש**:
- טבלת `business_hours` ב-DB (יום, שעת פתיחה, שעת סגירה)
- לוגיקה ב-handler שבודקת את השעה הנוכחית (Asia/Jerusalem)
- הודעת פתיחה שונה מחוץ לשעות הפעילות
- לוח חגים ישראלי (שבתות, חגים יהודיים, יום העצמאות וכו')

**התנהגות מוצעת**:
```
# בשעות הפעילות:
"שלום! 👋 ברוכים הבאים. איך אפשר לעזור?"

# מחוץ לשעות הפעילות:
"שלום! 🌙 אנחנו כרגע סגורים.
שעות הפעילות שלנו: א'-ה' 09:00-20:00, ו' 09:00-14:00.
אתם מוזמנים לשאול שאלות ואני אענה, או לקבוע תור ליום הבא."

# בערב חג:
"חג שמח! 🎉 נחזור לפעילות ביום X.
בינתיים, השאירו הודעה ונחזור אליכם."
```

**מימוש**:
- הוספת שדות `business_hours` ו-`holidays` ל-config או ל-DB
- פונקציית `is_business_open()` שמחזירה `bool` + הודעה מתאימה
- שילוב ב-`start_command` וב-`message_handler`

**ערך ללקוח**: מקצועיות. הלקוח מרגיש שהעסק מסודר ושקוף.

## 4. שליחת הודעות יזומות (Broadcast)

> **הנמקה**: בעל עסק קטן רוצה להודיע ללקוחות על מבצע, שינוי בשעות פעילות, או חידוש שירות. כרגע אין שום דרך לעשות את זה דרך הפאנל.

**מה נדרש**:
- דף חדש בפאנל הניהול: "שליחת הודעה לכולם"
- שליפת רשימת כל ה-`user_id` שהיו בשיחה עם הבוט
- שליחת הודעה לכל אחד מהם דרך Telegram API
- אפשרות לסנן: כל המשתמשים / רק מי שקבע תור / רק מי שפנה בחודש האחרון

**ממשק בפאנל**:
```
┌────────────────────────────────────────────────┐
│  📢 שליחת הודעה                                │
│                                                │
│  לקהל:  ○ כל הלקוחות (127)                     │
│         ○ לקוחות שקבעו תור (43)                 │
│         ○ לקוחות מהחודש האחרון (68)              │
│                                                │
│  הודעה:                                        │
│  ┌──────────────────────────────────────────┐   │
│  │ 🎉 חדשות! שירות חדש — החלקה ברזילאית    │   │
│  │ במחיר השקה! לפרטים כתבו "החלקה"         │   │
│  └──────────────────────────────────────────┘   │
│                                                │
│  [📤 שלח עכשיו]                                │
└────────────────────────────────────────────────┘
```

**חשוב**:
- הגבלת קצב שליחה (למניעת חסימה ע"י Telegram)
- שמירת לוג של הודעות שנשלחו
- אפשרות "תצוגה מקדימה" לפני שליחה

**ערך ללקוח**: הבוט הופך מכלי תגובתי (Reactive) לכלי יזום (Proactive). בעל עסק יכול לעדכן לקוחות בלחיצה.

## 5. קובץ יומן לאישור תור (.ics)

> **הנמקה**: אחרי שלקוח קובע תור, הוא צריך לזכור. שליחת קובץ `.ics` מוסיפה את התור אוטומטית ליומן הטלפון — עם תזכורת מובנית.

**מה נדרש**:
- יצירת קובץ `.ics` (פורמט iCalendar) עם פרטי התור
- שליחת הקובץ כמסמך בטלגרם אחרי אישור התור
- הקובץ כולל: שם שירות, תאריך, שעה, כתובת העסק, תזכורת שעה לפני

**דוגמה לקובץ `.ics`**:
```
BEGIN:VCALENDAR
BEGIN:VEVENT
SUMMARY:תספורת — מכון היופי של דנה
DTSTART:20260225T140000
DTEND:20260225T150000
LOCATION:דיזנגוף 50, תל אביב
DESCRIPTION:תור לתספורת גברים
BEGIN:VALARM
TRIGGER:-PT1H
DESCRIPTION:תזכורת: יש לך תור בעוד שעה!
END:VALARM
END:VEVENT
END:VCALENDAR
```

**מימוש**: ספריית `icalendar` ב-Python (קלת משקל). הקובץ נוצר ב-memory ונשלח כ-`document` ב-Telegram Bot API.

**ערך ללקוח**: הפחתת no-shows. הלקוח מוסיף ליומן בלחיצה אחת — עם תזכורת אוטומטית. לא תלוי במערכת תזכורות חיצונית.

## 6. QR Code להתחלת שיחה עם הבוט

> **הנמקה**: בעל עסק רוצה לשים סטיקר ליד הקופה, על כרטיס הביקור, או בפוסט באינסטגרם — שלקוחות יסרקו ויגיעו ישר לבוט.

**מה נדרש**:
- יצירת QR Code שמפנה ל-`https://t.me/BOT_USERNAME`
- הצגת ה-QR Code בפאנל הניהול (עם אפשרות הורדה כ-PNG)
- אפשרות לעיצוב מותאם (צבע, לוגו של העסק במרכז)

**מימוש**: ספריית `qrcode` ב-Python. ה-QR נוצר פעם אחת ונשמר כתמונה.

**שימושים**:
- סטיקר פיזי על הדלפק / דלת הכניסה
- חתימת מייל
- כרטיס ביקור דיגיטלי
- סטורי באינסטגרם / פוסט בפייסבוק

**ערך ללקוח**: מגשר בין העולם הפיזי לדיגיטלי. לקוח שנמצא פיזית בעסק יכול מיד להתחבר לבוט ולקבוע תור הבא.

## 7. כרטיס ביקור דיגיטלי (vCard)

> **הנמקה**: כשלקוח מבקש פרטי קשר, עדיף לשלוח כרטיס ביקור דיגיטלי שנשמר ישר באנשי הקשר של הטלפון — ולא רק טקסט שמתגלגל בצ'אט.

**מה נדרש**:
- יצירת קובץ `.vcf` עם פרטי העסק (שם, טלפון, כתובת, אתר, שעות)
- כפתור חדש או תוספת לכפתור המיקום: "📇 שמור אותנו באנשי הקשר"
- שליחת הקובץ כמסמך בטלגרם

**דוגמה לקובץ `.vcf`**:
```
BEGIN:VCARD
VERSION:3.0
FN:מכון היופי של דנה
ORG:מכון היופי של דנה
TEL;TYPE=WORK:03-1234567
ADR;TYPE=WORK:;;דיזנגוף 50;תל אביב;;6433001;ישראל
URL:https://dana-beauty.co.il
NOTE:א'-ה' 09:00-20:00 | ו' 09:00-14:00
END:VCARD
```

**ערך ללקוח**: הלקוח שומר את העסק באנשי הקשר. בפעם הבאה שצריך תור — הוא פשוט מחפש בטלפון. שיעור חזרה גבוה יותר.

## 8. ניווט בלחיצה — Waze ו-Google Maps

> **הנמקה**: כשהבוט שולח כתובת כטקסט, הלקוח צריך להעתיק ולהדביק ב-Waze. זה מסורבל. קישור ישיר לניווט חוסך את זה.

**מה נדרש**:
- שמירת קואורדינטות GPS של העסק (latitude, longitude) ב-config
- כשלקוח מבקש מיקום — שליחת Inline Keyboard עם כפתורי ניווט:

```
📍 דיזנגוף 50, תל אביב

[🗺️ פתח ב-Google Maps]  [🚗 נווט עם Waze]
```

**מימוש**:
```python
waze_url = f"https://waze.com/ul?ll={lat},{lng}&navigate=yes"
gmaps_url = f"https://www.google.com/maps/dir/?api=1&destination={lat},{lng}"
```
בנוסף — שליחת `send_location` של Telegram (מפה אינטראקטיבית בתוך הצ'אט).

**ערך ללקוח**: לקוח לוחץ כפתור אחד ו-Waze נפתח עם ניווט ישיר. חוויה חלקה, אפס חיכוכים.

## 9. הצעת שאלות המשך חכמות

> **הנמקה**: אחרי שהבוט עונה, הלקוח לפעמים לא יודע מה עוד אפשר לשאול. הצעת שאלות המשך רלוונטיות מגבירה אינטראקציה ומוציאה ערך מהבוט.

**מה נדרש**:
- לאחר כל תשובה, הבוט מציע 2-3 שאלות המשך כ-Inline Buttons
- השאלות נגזרות מההקשר (שירות, קטגוריה, Intent)

**דוגמאות**:

```
לקוח: "כמה עולה תספורת גברים?"
בוט:   "תספורת גברים עולה 80 ₪ ..."
       [📅 לקבוע תור]  [💇 שירותים נוספים]  [📍 איך מגיעים?]
```

```
לקוח: "מה שעות הפתיחה?"
בוט:   "שעות הפעילות שלנו: א'-ה' ..."
       [📅 לקבוע תור]  [📋 מחירון]  [📍 ניווט לעסק]
```

**מימוש**: מיפוי סטטי של intent → שאלות מוצעות. לא דורש קריאת LLM נוספת.

**ערך ללקוח**: שיעור המרה גבוה יותר. הלקוח "מתגלגל" מטבעי מעניין לפעולה (שאלה → תור).

## 10. התאמה אישית של אישיות הבוט

> **הנמקה**: מכון יופי מצפה לטון חם וידידותי. משרד עורכי דין מצפה לטון רשמי ומקצועי. כרגע הטון קבוע ב-System Prompt ודורש עריכת קוד.

**מה נדרש**:
- הגדרת "אישיות" בפאנל הניהול — בחירה מתוך תבניות מוכנות:

| אישיות | טון | מתאים ל- |
|---------|------|-----------|
| חם וידידותי | "היי! מה נשמע?" / אימוג'ים | סלון יופי, קפה, חנות בגדים |
| מקצועי ואדיב | "שלום, איך אוכל לסייע?" | מרפאה, עו"ד, רו"ח |
| צעיר ודינמי | "יו! מה קורה?" / סלנג | סטודיו לקעקועים, חנות סקייט |
| רשמי | "שלום רב. נשמח לעמוד לרשותכם." | משרד ממשלתי, בנק |

**מימוש**:
- שדה `bot_personality` ב-config/DB עם ערך enum
- תבניות System Prompt מותאמות שנטענות לפי הבחירה
- אפשרות לבעל העסק לכתוב "הנחיה מותאמת" בטקסט חופשי

**ערך ללקוח**: הבוט מרגיש כמו חלק אורגני מהעסק, לא כמו מוצר גנרי.

## 11. התאמה אישית של כפתורי התפריט

> **הנמקה**: לא כל עסק צריך את אותם כפתורים. מסעדה לא צריכה "קביעת תור" — היא צריכה "הזמנת שולחן". עו"ד צריך "תיאום פגישה" ולא "מחירון".

**מה נדרש**:
- הגדרת כפתורים בפאנל הניהול (תווית, פעולה, אייקון)
- כפתורי ברירת מחדל שניתן לערוך, להסתיר, או להחליף
- תמיכה בעד 6 כפתורים (3 שורות × 2)

**דוגמאות לפי סוג עסק**:

| עסק | כפתורים |
|------|---------|
| סלון יופי | 📋 מחירון, 📅 קביעת תור, 📍 ניווט, 👤 נציג |
| מסעדה | 🍽️ תפריט, 📅 הזמנת שולחן, 📍 ניווט, 📞 התקשרו |
| עו"ד | ⚖️ תחומי שירות, 📅 תיאום פגישה, 📞 צור קשר, 👤 נציג |
| חנות | 🛍️ קטלוג, 🚚 משלוחים, 📞 שירות לקוחות, 📍 סניפים |

**מימוש**: טבלת `menu_buttons` ב-DB. ה-handler טוען את הכפתורים מה-DB במקום מקבועים ב-Python.

**ערך ללקוח**: העסק מרגיש שהבוט נבנה בדיוק בשבילו, לא "אחד לכולם".

## 12. מצב חופשה / חירום

> **הנמקה**: בעל עסק לוקח חופשה של שבוע. הוא לא רוצה שלקוחות יקבעו תורים שלא יתקיימו. הוא צריך "מצב חופשה" שמעדכן אוטומטית.

**מה נדרש**:
- כפתור toggle בפאנל הניהול: "מצב חופשה" (פעיל/לא פעיל)
- הגדרת טווח תאריכים (מ-X עד-Y)
- הודעה מותאמת אישית
- בזמן מצב חופשה:
  - הבוט עונה על שאלות מידע (RAG) כרגיל
  - כפתור "קביעת תור" → "אנחנו בחופשה עד X, ניתן לקבוע תור החל מ-Y"
  - כפתור "דברו עם נציג" → "אנחנו בחופשה עד X. נחזור אליכם כשנחזור"

**דוגמת תגובה**:
```
"🏖️ אנחנו בחופשה מ-25.2 עד 2.3.
הבוט שלנו ממשיך לענות על שאלות!
לקביעת תור — נחזור ביום שני 3.3."
```

**ערך ללקוח**: שקט נפשי. בעל העסק לוקח חופשה בלי חרדה שלקוחות "נשרים".

## 13. פרופילי צוות ובחירת נותן שירות

> **הנמקה**: "אני רוצה תור אצל דנה, לא אצל עדי". לקוחות חוזרים רוצים את האדם הספציפי. כרגע אין אפשרות לציין העדפה.

**מה נדרש**:
- טבלת `staff` ב-DB (שם, תפקיד, תיאור קצר, שירותים מוצעים)
- שלב נוסף בתהליך קביעת התור: "עם מי תרצו?" (או "אין העדפה")
- הצגת פרופילי הצוות בתשובות רלוונטיות

**זרימת קביעת תור מורחבת**:
```
בוט: "עם מי תרצו לקבוע?"
     [דנה — מומחית צבע] [עדי — תספורות] [אין העדפה]
לקוח: "דנה"
בוט: "מעולה! באיזה תאריך?"
```

**מימוש**: שדה `preferred_staff` ב-טבלת `appointments`. שלב BOOKING_STAFF חדש ב-ConversationHandler.

**ערך ללקוח**: חוויה אישית. הלקוח מרגיש שהבוט מכיר את העסק מבפנים.

## 14. גלריית עבודות (פורטפוליו)

> **הנמקה**: עסקי יופי, עיצוב, ואומנות חיים ונושמים תמונות. "תראו מה עשינו" הוא כלי מכירה חזק. כרגע אין דרך להציג תמונות דרך הבוט.

**מה נדרש**:
- טבלת `portfolio` ב-DB (תיאור, URL לתמונה, קטגוריה, תאריך)
- כפתור "🖼️ גלריה" או שאלה "תראו לי דוגמאות לבליאז'"
- שליחת Media Group (אלבום תמונות) דרך Telegram API

**דוגמה**:
```
לקוח: "יש לכם דוגמאות של בליאז'?"
בוט:  "הנה כמה עבודות שלנו:"
      [📸 תמונה 1] [📸 תמונה 2] [📸 תמונה 3]
      "רוצים לקבוע תור לבליאז'? 📅"
```

**ניהול בפאנל**: דף ייעודי לעליית תמונות עם תיאור וקטגוריה.

**ערך ללקוח**: הבוט לא רק מוכר — הוא מראה. "תמונה שווה אלף מילים" ובמיוחד בתעשיית היופי.

## 15. פולו-אפ אחרי שירות

> **הנמקה**: 24 שעות אחרי הביקור, לשלוח הודעה קצרה: "איך היה? הכול בסדר?". זה בונה קשר, מזהה בעיות מוקדם, ומעודד חזרה.

**מה נדרש**:
- Scheduler (APScheduler) שרץ ברקע
- כשתור מסומן כ-"confirmed" ותאריכו עבר — שולח פולו-אפ
- הודעת ברירת מחדל עם אפשרות התאמה

**דוגמת הודעה**:
```
"היי [שם]! 👋
תודה שביקרתם אצלנו אתמול.
מקווים שנהניתם! אם יש משהו שנוכל לשפר — נשמח לשמוע.
לקביעת תור הבא: /start"
```

**מימוש**: Job ב-APScheduler שרץ כל שעה, בודק תורים שהושלמו ב-24 השעות האחרונות, ושולח הודעה למי שעוד לא קיבל.

**ערך ללקוח**: לקוחות מרגישים שאכפת מהם. שיעור חזרה עולה. בעיות מזוהות לפני שהופכות לביקורת שלילית ב-Google.

## 16. עידוד ביקורות ב-Google

> **הנמקה**: ביקורות Google הן הזהב של עסקים קטנים. כל ביקורת חיובית = לקוח חדש פוטנציאלי. אבל לקוחות שוכחים לכתוב. תזכורת עדינה עושה פלאים.

**מה נדרש**:
- קישור ישיר לכתיבת ביקורת ב-Google Business (place ID)
- שליחת הודעה 48 שעות אחרי ביקור (אחרי הפולו-אפ הראשוני)
- רק ללקוחות שהגיבו חיובית לפולו-אפ, או לא הגיבו כלל

**דוגמת הודעה**:
```
"שמחים שנהניתם! 😊
אם יש לכם רגע, נשמח לביקורת קצרה ב-Google — זה ממש עוזר לנו:
[⭐ כתבו ביקורת]"
```

**ערך ללקוח**: דירוג Google עולה → יותר לקוחות חדשים מחיפוש אורגני. ROI ישיר וברור.

## 17. מערכת הפניות (Referral)

> **הנמקה**: "חבר שלי סיפר לי על המקום". שיווק מפה לאוזן הוא הכי אפקטיבי לעסקים קטנים. מתן תמריץ הופך את זה לשיטתי.

**מה נדרש**:
- כל לקוח מקבל "קוד הפניה" ייחודי
- כשלקוח חדש מזכיר קוד הפניה (או לוחץ קישור ייעודי) → שניהם מקבלים הטבה
- מעקב בפאנל: מי הפנה, כמה הפניות הצליחו

**זרימה**:
```
בוט: "רוצים לשתף חבר? 🎁
      שלחו לו את הקישור הזה: t.me/DanaBeautyBot?start=ref_ABC123
      כשהוא יקבע תור — שניכם תקבלו 10% הנחה!"
```

**ערך ללקוח**: ערוץ גיוס לקוחות חדש בעלות אפסית. הלקוחות הקיימים עושים את השיווק.

## 18. Telegram Mini App לחוויה עשירה

> **הנמקה**: Telegram תומך ב-Mini Apps — אפליקציות ווב שנפתחות בתוך הצ'אט. זה מאפשר חוויות שלא ניתן לממש בטקסט: לוח שנה ויזואלי, גלריה, טופס מורכב.

**תרחישים לשימוש**:

| תרחיש | חוויה ב-Mini App |
|--------|------------------|
| קביעת תור | לוח שנה ויזואלי עם חלונות פנויים |
| מחירון | טבלה מעוצבת עם תמונות לכל שירות |
| גלריה | גריד תמונות swipeable |
| טופס פרטים | טופס מלא עם ולידציה בצד לקוח |

**מימוש**:
- דף HTML + JS שנטען מהשרת (Flask כבר קיים)
- כפתור `WebAppInfo` ב-Telegram Bot API
- תקשורת עם ה-backend דרך REST API

**ערך ללקוח**: חוויית משתמש ברמה של אפליקציה מובייל — ללא צורך בהורדת אפליקציה.

## 19. תגובות מדיה עשירה (תמונות, PDF, וידאו)

> **הנמקה**: כשלקוח שואל "מה המחירון?" — עדיף לשלוח PDF מעוצב של המחירון ולא רק טקסט ארוך. כשהוא שואל "איפה אתם?" — אפשר לשלוח תמונה של החזית.

**מה נדרש**:
- שדה `media_type` ו-`media_url` בטבלת `kb_entries`
- כשתשובת RAG מגיעה מרשומה עם מדיה מצורפת — הבוט שולח גם את המדיה
- תמיכה בסוגי מדיה: תמונה, PDF, וידאו קצר

**דוגמאות**:

| שאלה | תגובה |
|-------|--------|
| "מה המחירון?" | טקסט מקוצר + 📎 מחירון.pdf |
| "איפה אתם?" | כתובת + 📸 תמונת חזית העסק + 📍 מיקום |
| "יש הדרכה לטיפוח?" | "בוודאי!" + 🎥 סרטון 30 שניות |

**ניהול בפאנל**: אפשרות להעלות קובץ בעת יצירת/עריכת רשומת KB.

**ערך ללקוח**: מחירון PDF מעוצב מרגיש מקצועי. תמונה של העסק בונה אמון. סרטון מייצר עניין.
